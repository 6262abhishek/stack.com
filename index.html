What is a Stack?

A Stack is a linear data structure that follows the LIFO (Last In, First Out) principle. This means the last element inserted is the first one to be removed.

LIFO Principle

Imagine a stack of plates. When you add a new plate to the top, that's the "push" operation. When you remove a plate, you take it from the top - that's the "pop" operation. The last plate you added is the first one you remove.

Key Characteristics

  • Elements are added and removed from the same end (top)
  • Follows Last In First Out (LIFO) order
  • Supports four basic operations: push, pop, peek, and isEmpty
  • Can be implemented using arrays, linked lists, or other data structures

Common Stack Operations

Stacks support several fundamental operations that allow you to manipulate and access data efficiently.

Operation Description Example
push(x) Add (insert) an element on the top of the stack push(10)
pop() Remove the top element from the stack pop() → removes last element
peek() / top() View the top element without removing it peek() → returns top
isEmpty() Check if the stack is empty true / false
size() Returns total number of elements returns count

class Stack { int top; int maxSize; int[] stackArray; // Constructor Stack(int size) { maxSize = size; stackArray = new int[maxSize]; top = -1; } // Push operation void push(int value) { if (top == maxSize - 1) { System.out.println("Stack Overflow! Cannot push " + value); } else { stackArray[++top] = value; System.out.println(value + " pushed into stack"); } } // Pop operation int pop() { if (top == -1) { System.out.println("Stack Underflow! Nothing to pop"); return -1; } else { int popped = stackArray[top--]; System.out.println(popped + " popped from stack"); return popped; } } // Peek operation int peek() { if (top == -1) { System.out.println("Stack is empty"); return -1; } else { return stackArray[top]; } } // Check if stack is empty boolean isEmpty() { return (top == -1); } // Check size int size() { return top + 1; } // Display elements void display() { if (isEmpty()) { System.out.println("Stack is empty"); } else { System.out.print("Stack elements: "); for (int i = 0; i <= top; i++) { System.out.print(stackArray[i] + " "); } System.out.println(); } } }

class Stack { private Node top; private int size; Stack() { this.top = null; this.size = 0; } // Push operation void push(int data) { Node newNode = new Node(data); newNode.next = top; // link new node to previous top top = newNode; // make new node the top size++; System.out.println(data + " pushed into stack"); } // Pop operation int pop() { if (isEmpty()) { System.out.println("Stack Underflow! Nothing to pop"); return -1; } int value = top.data; top = top.next; // move top to next node size--; System.out.println(value + " popped from stack"); return value; } // Peek operation int peek() { if (isEmpty()) { System.out.println("Stack is empty"); return -1; } return top.data; } // Check if stack is empty boolean isEmpty() { return top == null; } // Return size int size() { return size; } // Display stack elements void display() { if (isEmpty()) { System.out.println("Stack is empty"); return; } Node temp = top; System.out.print("Stack elements (Top to Bottom): "); while (temp != null) { System.out.print(temp.data + " "); temp = temp.next; } System.out.println(); } }

Real-Life Examples of Stack

Stacks are not just theoretical concepts - they're used in many real-world applications.

Undo/Redo in Text Editors

Every action (like typing) is pushed into a stack. Undo pops the last action. Redo pushes it again.

Browser History

Each visited page is pushed into a stack. When you press back, the top page is popped.

App Navigation (Back Button)

Screens are stacked — pressing back removes (pops) the top screen.

Recursion in Programming

Function calls are stored in the call stack; the last called function returns first.

Stack of Plates or Books

You can only take or add from the top — same concept as LIFO.

import java.util.LinkedList; import java.util.Queue; class StackUsingTwoQueues { Queue<Integer> q1 = new LinkedList<>(); Queue<Integer> q2 = new LinkedList<>(); // Push operation (costly) void push(int x) { // Step 1: Push element into q2 q2.add(x); // Step 2: Move all elements of q1 into q2 while (!q1.isEmpty()) { q2.add(q1.remove()); } // Step 3: Swap q1 and q2 Queue<Integer> temp = q1; q1 = q2; q2 = temp; System.out.println("Pushed " + x + " to stack"); } // Pop operation int pop() { if (q1.isEmpty()) { System.out.println("Stack Underflow! Nothing to pop"); return -1; } int popped = q1.remove(); System.out.println("Popped: " + popped); return popped; } // Peek operation int peek() { if (q1.isEmpty()) { System.out.println("Stack is empty"); return -1; } return q1.peek(); } // Check if empty boolean isEmpty() { return q1.isEmpty(); } // Display stack elements void display() { if (isEmpty()) { System.out.println("Stack is empty"); return; } System.out.print("Stack elements: "); for Integer item : q1) { System.out.print(item + " "); } System.out.println(); } } public class Main { public static void main(String[] args) { StackUsingTwoQueues stack = new StackUsingTwoQueues(); stack.push(10); stack.push(20); stack.push(30); stack.display(); System.out.println("Top element: " + stack.peek()); stack.pop(); stack.display(); System.out.println("Is stack empty? " + stack.isEmpty()); } }
Output
Pushed 10 to stack Pushed 20 to stack Pushed 30 to stack Stack elements: 30 20 10 Top element: 30 Popped: 30 Stack elements: 20 10 Is stack empty? false

Explanation

In this implementation, we use two queues to simulate a stack:

  • Push Operation (O(n)): We add the new element to q2, then move all elements from q1 to q2, and finally swap q1 and q2. This ensures the newest element is always at the front of q1.
  • Pop Operation (O(1)): We simply remove and return the front element of q1.
  • Peek Operation (O(1)): We return the front element of q1 without removing it.

import java.util.LinkedList; import java.util.Queue; class StackUsingSingleQueue { Queue<Integer> q = new LinkedList<>(); // Push operation (costly) void push(int x) { // Get current size of queue int size = q.size(); // Add the new element to the queue q.add(x); // Rotate the queue (size - 1) times for (int i = 0; i < size; i++) { q.add(q.remove()); } System.out.println("Pushed " + x + " to stack"); } // Pop operation int pop() { if (q.isEmpty()) { System.out.println("Stack Underflow! Nothing to pop"); return -1; } int popped = q.remove(); System.out.println("Popped: " + popped); return popped; } // Peek operation int peek() { if (q.isEmpty()) { System.out.println("Stack is empty"); return -1; } return q.peek(); } // Check if empty boolean isEmpty() { return q.isEmpty(); } // Display stack elements void display() { if (isEmpty()) { System.out.println("Stack is empty"); return; } System.out.print("Stack elements: "); for Integer item : q) { System.out.print(item + " "); } System.out.println(); } } public class Main { public static void main(String[] args) { StackUsingSingleQueue stack = new StackUsingSingleQueue(); stack.push(10); stack.push(20); stack.push(30); stack.display(); System.out.println("Top element: " + stack.peek()); stack.pop(); stack.display(); System.out.println("Is stack empty? " + stack.isEmpty()); } }
Output
Pushed 10 to stack Pushed 20 to stack Pushed 30 to stack Stack elements: 30 20 10 Top element: 30 Popped: 30 Stack elements: 20 10 Is stack empty? false

Explanation

In this implementation, we use only one queue to simulate a stack:

  • Push Operation (O(n)): We add the new element to the queue, then rotate the queue (size - 1) times to bring the new element to the front.
  • Pop Operation (O(1)): We simply remove and return the front element of the queue.
  • Peek Operation (O(1)): We return the front element of the queue without removing it.

Advanced Stack Applications

Stacks are used in many advanced computer science applications beyond the basic examples:

Expression Evaluation

Stacks are used to evaluate arithmetic expressions in postfix (Reverse Polish Notation) and prefix notation. The algorithm processes each token and uses the stack to store operands and intermediate results.

Syntax Parsing

Compilers use stacks to parse source code and check for balanced parentheses, brackets, and braces. When an opening bracket is encountered, it's pushed onto the stack; when a closing bracket is found, the stack is popped to check for a match.

Graph Algorithms

Depth-First Search (DFS) uses a stack (either explicitly or implicitly through recursion) to explore as far as possible along each branch before backtracking. This is fundamental in many graph algorithms.

Memory Management

The call stack is used to manage function calls in most programming languages. Each time a function is called, its context is pushed onto the stack, and when it returns, the context is popped. This is crucial for recursion and exception handling.

Undo/Redo Operations

Advanced implementations use two stacks: one for undo operations and another for redo operations. When an action is performed, it's pushed onto the undo stack. When undo is executed, the action is popped from the undo stack and pushed onto the redo stack.

Backtracking Algorithms

Algorithms that solve problems by trying all possible solutions use stacks to keep track of the current path. Examples include solving mazes, N-Queens problem, and other constraint satisfaction problems.

import java.util.Stack; public class PostfixEvaluation { public static int evaluatePostfix(String expression) { Stack<Integer> stack = new Stack<>(); for int i = 0; i < expression.length(); i++) { char c = expression.charAt(i); // If the character is a digit, push it to the stack if (Character.isDigit(c)) { stack.push(Character.getNumericValue(c))); } // If the character is an operator, pop two elements and apply the operator else { int val1 = stack.pop(); int val2 = stack.pop(); switch (c) { case '+': stack.push(val2 + val1); break; case '-': stack.push(val2 - val1); break; case '*': stack.push(val2 * val1); break; case '/': stack.push(val2 / val1); break; } } } return stack.pop(); } public static void main(String[] args) { String expression = "231*+9-"; System.out.println("Postfix expression: " + expression); System.out.println("Result: " + evaluatePostfix(expression)); } }
Output
Postfix expression: 231*+9- Result: -4

Explanation

The postfix expression "231*+9-" is evaluated as follows:

  • Push 2 onto the stack
  • Push 3 onto the stack
  • Push 1 onto the stack
  • Encounter *: Pop 1 and 3, calculate 3*1 = 3, push 3 onto the stack
  • Encounter +: Pop 3 and 2, calculate 2+3 = 5, push 5 onto the stack
  • Push 9 onto the stack
  • Encounter -: Pop 9 and 5, calculate 5-9 = -4, push -4 onto the stack
  • Final result is -4

Top Stack Interview Questions

Common questions about stacks asked in technical interviews with clear answers.

What is a Stack?

A Stack is a linear data structure that follows LIFO (Last In, First Out) order. Example: plates stack — last kept plate is removed first.

What are basic operations of Stack?

• push() → add item
• pop() → remove item
• peek() → get top item
• isEmpty() → check if empty
• size() → count items

Difference between Stack and Queue?

Stack: LIFO, push/pop, Top-based
Queue: FIFO, enqueue/dequeue, Front-rear based

Can a stack overflow or underflow occur?

• Overflow → When we push to a full array stack.
• Underflow → When we pop from an empty stack.
(LinkedList stack doesn't overflow since it grows dynamically.)

What is the time complexity of stack operations?

Push: O(1)
Pop: O(1)
Peek: O(1)
isEmpty: O(1)
All are constant time operations

Explain recursion and call stack.

Every function call is pushed onto the call stack. When a function ends, it is popped. This helps the program remember where to return after each function.

Where are stacks used in programming?

• Recursion and function calls
• Expression evaluation (postfix, prefix)
• Syntax parsing (compiler design)
• Undo/Redo operations
• Memory management (call stack)

What is Stack Overflow (in memory)?

When recursive calls go infinite (no stop condition), the program keeps pushing function calls into stack memory → StackOverflowError occurs in Java.

How to implement a stack using two queues?

Use two queues q1 and q2. For push operation, add new element to q2, then move all elements from q1 to q2, and finally swap q1 and q2. For pop operation, simply remove from q1.

How to implement a stack using a single queue?

Add the new element to the queue, then rotate the queue (size-1) times to bring the new element to the front. For pop operation, simply remove from the front.
